## 1. 專案概述

### 1.1 背景與動機

現代人飲食多元，常常在網路上看到喜歡的食譜卻找不到好的方式收藏、整理。傳統方式（書籤、截圖、筆記本）缺乏結構化管理，導致食譜散落各處、難以查找。本系統旨在提供一個集中管理、方便搜尋的食譜收藏平台，讓使用者能依照食材、難度等條件快速找到適合的料理方式。

### 1.2 目標用戶

| 角色 | 描述 |
|------|------|
| **一般用戶** | 喜愛烹飪或想學做菜的一般大眾，需要收藏、瀏覽與搜尋食譜 |
| **管理員** | 負責系統維護、審核食譜內容、管理用戶與分類的後台人員 |

### 1.3 核心價值主張

- **集中收藏**：所有食譜統一存放，不再散落各處
- **智慧搜尋**：依照食材組合找到可做的料理
- **難度分級**：讓不同程度的廚師都能找到適合自己的食譜
- **易於管理**：管理員可維護內容品質，確保資料準確性

## 2. 功能需求

### 功能列表與使用者故事

#### 功能 1：儲存食譜

**描述**：使用者可以新增並儲存個人食譜，包含食材、步驟、封面圖片等資訊。

**使用者故事**：
- 作為**一般用戶**，我希望能新增一道食譜並填寫食材與步驟，以便將喜愛的料理方式永久保存。
- 作為**一般用戶**，我希望能編輯或刪除我儲存的食譜，以便隨時更新內容。
- 作為**管理員**，我希望能管理所有用戶的食譜，以便審核並維護平台內容品質。

**所需欄位**：
- 食譜名稱（必填）
- 食材清單（食材名稱 + 份量）
- 烹飪步驟（有順序的文字說明）
- 難度係數（1–5 星）
- 烹飪時間（分鐘）
- 分類標籤（如：早餐、甜點、素食）
- 封面圖片（選填）

---

#### 功能 2：瀏覽食譜

**描述**：使用者可以在首頁或分類頁瀏覽所有公開食譜，支援分頁與分類篩選。

**使用者故事**：
- 作為**一般用戶**，我希望能在首頁看到最新或熱門食譜列表，以便快速發現感興趣的料理。
- 作為**一般用戶**，我希望能點擊食譜卡片進入詳細頁面，查看完整的食材與步驟。
- 作為**一般用戶**，我希望能依照分類（如素食、甜點）篩選食譜，以便找到特定類型的料理。

---

#### 功能 3：搜尋食譜

**描述**：使用者可以透過關鍵字搜尋食譜名稱或描述，快速找到目標食譜。

**使用者故事**：
- 作為**一般用戶**，我希望能在搜尋框輸入關鍵字（如「雞肉」、「義大利麵」），以便快速找到相關食譜。
- 作為**一般用戶**，我希望搜尋結果能即時顯示，不需要重新整理頁面，以便提升操作效率。

---

#### 功能 4：從食材組合搜尋食譜

**描述**：使用者可以輸入目前手邊有的食材，系統自動找出可以製作的食譜。

**使用者故事**：
- 作為**一般用戶**，我希望能輸入多個食材（如：雞蛋、番茄、洋蔥），系統推薦能用這些食材做的食譜，以便有效利用冰箱食材、減少浪費。
- 作為**一般用戶**，我希望系統能顯示「食材符合度」（例如：8/10 種食材符合），以便判斷哪道料理最適合現有食材。

#### 功能 5：難度係數

**描述**：每道食譜標示難度等級，使用者可依難度篩選食譜。

**使用者故事**：

- 作為**一般用戶**，我希望能在搜尋或瀏覽時依難度篩選，以便只看到我能挑戰的食譜。
- 作為**食譜作者**，我希望在新增食譜時能選擇難度評級，以便讓其他用戶正確評估這道料理的挑戰性。

**使用者故事**：

- 作為**管理員**，我希望能透過後台管理用戶帳號，以便處理違規內容或封鎖惡意用戶。

#### 功能 7：管理員後台

**描述**：管理員擁有專屬後台，可管理食譜內容、用戶帳號與分類標籤。

- 作為**管理員**，我希望能新增或修改分類標籤，以便讓食譜分類更準確。

## 3. 非功能需求

### 3.1 技術限制

| 項目 | 規格 |
|------|------|
| 後端框架 | Python Flask |
| 模板引擎 | Jinja2 |
| 資料庫 | SQLite |
| 前端 | HTML + CSS（Vanilla）+ JavaScript |
| 部署環境 | 本機開發 / 可選部署至雲端 |

### 3.2 效能考量

- 食譜列表頁需支援**分頁**，每頁最多顯示 12 筆，避免一次載入過多資料
- 食材組合搜尋需在 **2 秒內**回傳結果
- 圖片上傳需限制大小（建議最大 5MB），避免伺服器負擔過重

### 3.3 安全考量

- 使用者密碼需經過 **Hash 加密**（如 `bcrypt`）儲存，不得明文保存
- 所有表單需防範 **SQL Injection**（透過 ORM 或參數化查詢）
- 管理員頁面需進行**身份驗證**，未登入者不得存取
- 上傳圖片需檢查副檔名與 MIME Type，防止惡意檔案上傳
