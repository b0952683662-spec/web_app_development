# 產品需求文件 (PRD) - 食譜收藏夾

## 1. 專案概述

### 背景與動機
現代人常在網路上看到喜歡的食譜，但缺乏一個集中且易於管理的平台來收藏與分類。此外，烹飪時常遇到需要依據實際用餐人數調整食材份量的困擾。因此，開發一個專屬的「食譜收藏夾」網站，幫助使用者整合食譜資訊，並提供便利的烹飪輔助功能，如份量換算與分類管理。

### 目標用戶
- **一般用戶**：喜愛下廚、需要收藏與管理食譜的家庭主婦/夫、料理愛好者。
- **管理員**：負責管理系統中的公共食譜、維護分類標籤與平台秩序的人員。

### 核心價值主張
提供一站式的食譜管理服務，不僅能將材料與步驟清晰呈現，更支援動態份量換算與彈性分類，讓烹飪的前置作業變得輕鬆精準。

---

## 2. 功能需求

### 1. 新增食譜
- **作為** 一般用戶或管理員
- **我希望** 能夠新增一份食譜，並填寫名稱、圖片、簡介、材料、步驟與分類
- **以便** 將完整的烹飪資訊收藏與保存下來

### 2. 列出材料
- **作為** 準備料理的用戶
- **我希望** 能在食譜頁面清楚看到材料清單與對應的單位/份量
- **以便** 方便我進行採買與備料

### 3. 列出步驟
- **作為** 正在進行烹飪的用戶
- **我希望** 頁面能按照順序條列出烹飪步驟
- **以便** 我能在廚房按部就班地完成料理

### 4. 份量自動換算功能
- **作為** 需要依照用餐人數準備料理的用戶
- **我希望** 能夠輸入或調整目標份量（例如從 2 人份改成 4 人份），系統自動按比例換算所有材料的所需數量
- **以便** 準確準備食材，避免浪費或份量不足

### 5. 食譜分類
- **作為** 想要尋找特定類型食譜的用戶
- **我希望** 能夠透過分類標籤（如：中式、甜點、快速料理）來瀏覽或篩選食譜
- **以便** 快速找出符合當下需求的菜色

---

## 3. 非功能需求

### 技術限制
- 後端框架：Flask (Python)
- 模板引擎：Jinja2
- 資料庫：SQLite (適合輕量級應用與快速開發)

### 效能與安全考量
- **效能**：頁面載入時間需小於 3 秒；需考慮食譜圖片的處理，避免檔案過大導致載入緩慢。
- **安全**：支援基本的使用者認證，防止未授權操作；表單需具備防範 SQL Injection 與 XSS 攻擊的基本防護。

---

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必須要有)
- 新增與檢視單一食譜功能（包含標題、簡介、材料、步驟）。
- 食譜的清單列表。
- 基本的 SQLite 資料庫儲存，確保資料不會遺失。

### Should Have (應該要有)
- 份量自動換算功能（此專案的重點亮點）。
- 基礎食譜分類與篩選功能。

### Nice to Have (有了更好)
- 使用者註冊與登入功能（區分「我的收藏」與「公共區」）。
- 管理員後台介面（管理所有使用者與內容）。
- 圖文並茂的步驟與分享功能。
